# BLP Liminal — 디자인 언어 스펙

> **핵심 시드:** 하프톤(Halftone) × 파스텔톤(Pastel) × 몽환적(Dreamlike)  
> **철학:** 인쇄된 현실의 경계에서 디지털 잔상이 출몰하는 공간  
> **컬러 시스템:** BLP (Blue Light Palette) — 43색 전체 자율 사용  
> **타이포그래피:** 한국어 중심 멀티 폰트 스택  
> **핵심 원칙:** 사용자의 행동과 무관하게, 가만히 있어도 불안정해 보인다

---

## 1. 언어의 철학

**BLP Liminal**은 "인쇄된 꿈"을 전제로 한다. 모든 것이 평평한 2D 표면 위에 놓여 있지만, 그 표면은 언제든지 뒤틀릴 수 있는 불안정한 막이다.

**가장 중요한 원칙:** 이 불안정성은 사용자가 스크롤하거나 클릭할 때만 발생하는 것이 아니다. **가만히 있어도** 아날로그 도트와 디지털 글리치는 예측 불가능한 순간에 서로를 침범한다. BLP 팔레트의 43색은 고정된 역할에 얽매이지 않고, 필요에 따라 자율적으로 대립하고 침투한다.

이 언어는 **보편적 매체**를 목표로 하며, 웹/앱/인쇄/게임 UI를 가리지 않고 동일한 질감을 유지할 수 있어야 한다.

### 2.9 자율 사용 원칙

- **고정 매핑 없음:** 어떤 색이 "메인 컬러"인지 미리 정하지 않는다
- **랜덤 할당:** 글리치 발동 시, 43색 중 랜덤하게 2~3색이 선택되어 충돌한다
- **지속적 드리프트:** 배경, 도트, 블록의 색상은 6~14초 간격으로 미세하게 이동한다
- **대비 규칙:** 인접한 두 색상은 채도가 비슷하거나, 명도가 극명하게 차이나도록 선택한다

### 3.2 자동 전환 메커니즘 (Auto Morphing)

**스크롤 속도나 마우스 위치에 의존하지 않는다.** 시간과 랜덤성에 의해 폰트가 자동으로 전환된다.

```javascript
// 순수 시간 기반 자동 전환
function autoTypoMorph() {
  const phase = Math.random();
  if (phase > 0.6) {
    // 40%: Liminal (BookkMyungjo)
    elements.forEach(el => el.classList.remove('digital-phase', 'glitch-phase'));
  } else if (phase > 0.25) {
    // 35%: Digital (GoormSans)
    elements.forEach(el => {
      el.classList.remove('glitch-phase');
      el.classList.add('digital-phase');
    });
  } else {
    // 25%: Glitch (SlowGothic) — 0.8~2초간 유지
    elements.forEach(el => {
      el.classList.remove('digital-phase');
      el.classList.add('glitch-phase');
    });
    setTimeout(() => {
      elements.forEach(el => el.classList.remove('glitch-phase'));
    }, 800 + Math.random() * 1200);
  }
  setTimeout(autoTypoMorph, 4000 + Math.random() * 8000); // 4~12초 간격
}
```

```css
.typo-morph {
  font-family: 'BookkMyungjo', serif;
  transition: font-family 2.5s cubic-bezier(0.4, 0, 0.2, 1),
              letter-spacing 2.5s ease,
              font-weight 2.5s ease,
              opacity 2.5s ease,
              color 3s ease;
}

.typo-morph.digital-phase {
  font-family: 'GoormSans', sans-serif;
  letter-spacing: -0.02em;
  font-weight: 300;
  opacity: 0.85;
}

.typo-morph.glitch-phase {
  font-family: 'SlowGothic', sans-serif;
  letter-spacing: 0.04em;
  font-weight: 700;
  text-shadow: 2px 0 #00D620, -2px 0 #DBBDBD;
  animation: text-flicker 0.3s steps(2) infinite;
}
```

## 4. 시각 시스템 (Visual System)

### 4.1 하프톤: 실시간 Canvas 렌더링

#### 알고리즘
```javascript
const cellSize = 10;        // 기본 셀 크기 (지속적으로 9~11px로 미세 변동)
const maxR = cellSize / 2 - 0.8;
const renderThreshold = 0.4;

function renderHalftone(dotColor, bgColor) {
  ctx.fillStyle = bgColor;  // BLP 팔레트 중 랜덤
  ctx.fillRect(0, 0, w, h);
  ctx.fillStyle = dotColor; // BLP 팔레트 중 랜덤 (주로 Dark 계열)

  for (let y = 0; y < h; y += cellSize) {
    for (let x = 0; x < w; x += cellSize) {
      const brightness = ...; // 픽셀 평균 명도
      const radius = maxR * (1 - brightness / 255);
      if (radius > renderThreshold) {
        ctx.arc(x + cellSize/2, y + cellSize/2, radius, 0, Math.PI * 2);
        ctx.fill();
      }
    }
  }
}
```

#### 지속적 미세 불안정성 (Micro Instability)
```javascript
function microInstability() {
  cellSize = 9 + Math.random() * 2;        // 9~11px 사이 미세 변동
  canvas.style.opacity = 0.28 + Math.random() * 0.08;  // 0.28~0.36
  currentDotColor = randomDarkBLPColor();   // #00193D, #0026A3, #005BDD 등
  renderHalftone();
  setTimeout(microInstability, 2000 + Math.random() * 4000); // 2~6초
}
```

### 4.2 글리치 오버레이 (Glitch Overlay)

```css
.glitch-overlay {
  position: fixed; inset: 0; pointer-events: none; z-index: 9999;
  opacity: 0; transition: opacity 0.08s steps(3);
  mix-blend-mode: overlay;

```

#### 랜덤성 규칙 (강화됨)
```javascript
function triggerGlitch() {
  const rand = Math.random();
  if (rand > 0.95) {
    // 5%: Full Collapse (2~4초)
    // — Canvas 도트가 랜덤 BLP 색상으로 전환
    // — Glitch overlay가 2색의 repeating-conic-gradient로 붕괴
    // — solid-bg, pastel-block 모두 랜덤 BLP 색상으로 변환
    // — canvas filter: contrast(1.6) saturate(1.8) hue-rotate(랜덤)
  } else if (rand > 0.80) {
    // 15%: Glitch Intrusion (0.5~2초)
    // — 랜덤 BLP 색상의 conic-gradient 번쩍임
    // — Canvas 도트색이 동일 색상으로 동기화
  } else if (rand > 0.55) {
    // 25%: Ambient Drift
    // — solid-bg, pastel-block, divider가 랜덤 BLP 색상으로 이동
  }
  // 45%: 평온 (Nothing)
  scheduleNextGlitch();
}

function scheduleNextGlitch() {
  setTimeout(triggerGlitch, 1500 + Math.random() * 6000); // 1.5~7.5초
}
```

### 4.3 공간 구성: 평평한 2D + 리미널 루프

**깊이는 없다.** 반복되는 선과 블록이 무한한 공간감을 환각처럼 유발한다.

```css
    repeating-linear-gradient(0deg, transparent 0px, transparent 160px,
      rgba(219,189,189,0.03) 160px, rgba(219,189,189,0.03) 161px);
}
```

### 4.4 모션: 느린 유동 + 도트의 호흡

```css

}
@keyframes canvas-breathe {
  0%, 100% { transform: translate(0,0) scale(1); opacity: 0.32; }
  25% { transform: translate(0.5px,-0.8px) scale(1.008); opacity: 0.35; }
  50% { transform: translate(-0.3px,0.6px) scale(0.995); opacity: 0.28; }
  75% { transform: translate(0.6px,0.3px) scale(1.004); opacity: 0.34; }

```

- **Pastel Block 드리프트:** 6~14초 간격으로 위치와 색상이 미세하게 이동
- **Hero Accent 드리프트:** 8~18초 간격으로 `<em>`과 dot의 색상이 랜덤 BLP 색상으로 변환
- **모션 금지:** bounce, spring, 빠른 이동은 절대 없음

## 5. 상호작용 원칙 (Interaction Principles)

### 5.1 자동 불안정성 (Auto Instability)

| 상태 | 발생 빈도 | 현상 |
|------|-----------|------|
| **평온** | 45% | 아무 일도 없음. 도트가 숨을 쉰다 |
| **미세 불안정** | 25% | cellSize 미세 변동, dot 색상이 어두운 BLP 색으로 전환, canvas opacity 변동 |
| **글리치 침입** | 15% | 랜덤 BLP 색상의 conic-gradient 번쩍임, 0.5~2초 |
| **완전 붕괴** | 5% | 전체 화면이 2~4초간 랜덤 BLP 색상으로 재구성, 후 평온 복귀 |
| **앰비언트 드리프트** | 10% (백그라운드) | solid-bg, pastel-block, divider가 랜덤 색상으로 느리게 이동 |

### 5.2 사용자 행동의 영향 (최소화)

- **스크롤:** 타이포그래피 전환에 영향 없음 (순수 시간 기반)
- **클릭/탭:** 30% 확률로 국소적 글리치 (선택적)
- **호버:** Works 리스트 항목에 40% 확률로 해당 타이틀이 Glitch 폰트로 0.6초 변형
- **오래 머무름:** 30초 이상 시 화면 가장자리에 랜덤 BLP 색상의 inset box-shadow 침식

## 6. 구현 가이드 (Implementation Guide)

### 6.1 필수 기술 스택

- **2D Canvas:** 실시간 하프톤 (getImageData + arc)
- **CSS:** Grid/Flexbox, CSS Variables, mix-blend-mode, transition
- **JS:** 랜덤 타이머 체인, BLP 43색 배열, 폰트 전환 로직
- **폰트:** CDN 직접 로드 (woff2, font-display: swap)

### 6.2 성능 고려사항

- Canvas `alpha: false`, offscreen `willReadFrequently: true`
- DPR 제한: `Math.min(window.devicePixelRatio, 2)`
- 글리치는 국소적 요소에만 적용
- `requestAnimationFrame` 기반 렌더링

### 6.3 접근성 (A11y)

- `prefers-reduced-motion` 시 모든 애니메이션/전환/글리치 비활성화
- `aria-live="polite"`로 글리치 알림 (과도한 알림 방지)
- 텍스트/배경 대비는 4.5:1 이상 유지 (BLP DARK #00193D vs BLP WHITE #FAFCFF = 15.8:1)
